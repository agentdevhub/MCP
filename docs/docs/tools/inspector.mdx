[MCP Inspector](https://github.com/modelcontextprotocol/inspector) 是一款用于测试和调试 MCP 服务器的交互式开发者工具。虽然[调试指南](/docs/tools/debugging)已将 Inspector 作为整体调试工具集的一部分进行介绍，本文档将深入解析该工具的各项功能特性。




```bash
npx @modelcontextprotocol/inspector <command>
```

```bash
npx @modelcontextprotocol/inspector <command> <arg1> <arg2>
```

## 快速入门

### 安装与基础使用

无需安装即可通过 `npx` 直接运行 Inspector：


  ```bash
  npx -y @modelcontextprotocol/inspector npx <package-name> <args>
  # For example
  npx -y @modelcontextprotocol/inspector npx server-postgres postgres://127.0.0.1/testdb
  ```
#### 检测 NPM 或 PyPi 上的服务器

这是启动 [NPM](https://npmjs.com) 或 [PyPi](https://pypi.com) 服务器包的常用方式：
  ```bash
  npx @modelcontextprotocol/inspector uvx <package-name> <args>
  # For example
  npx @modelcontextprotocol/inspector uvx mcp-server-git --repository ~/code/mcp/servers.git
  ```

对于本地开发或通过仓库下载的服务器，最常用的检测方式是：
  ```bash
  npx @modelcontextprotocol/inspector \
    uv \
    --directory path/to/server \
    run \
    package-name \
    args...
  ```

Inspector 提供以下核心功能与您的 MCP 服务器进行交互：

### 服务器连接面板

- 选择连接服务器使用的[传输协议](/docs/concepts/transports)
- 支持本地服务器的命令行参数和环境变量自定义

### 资源标签页
- 列出所有可用资源

- 显示资源元数据（MIME 类型、描述信息）
- 支持资源内容审查
- 提供订阅测试功能

### 提示词标签页

- 展示可用提示词模板
- 显示提示参数及其描述
- 支持自定义参数测试提示词

- 预览生成的消息内容



### 工具标签页
- 列出可用工具集
- 显示工具模式及功能描述
- 支持自定义输入测试工具

- 展示工具执行结果

### 通知面板
- 呈现服务器记录的所有日志
- 显示来自服务器的实时通知


## 最佳实践
### 开发工作流
1. 启动开发
   - 通过 Inspector 启动服务器
   - 验证基础连接性

   - 检查能力协商过程


2. 迭代测试
   - 修改服务器代码
   - 重新构建服务器
   - 重新连接 Inspector
   - 测试受影响功能
   - 监控消息流


3. 边界测试
   - 无效输入测试
   - 缺失提示参数测试
   - 并发操作验证
   - 错误处理与响应检查
