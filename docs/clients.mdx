## 客户端详情

### 5ire
[5ire](https://github.com/nanbingxyz/5ire) 是一个开源跨平台桌面AI助手，支持通过MCP服务器使用工具。

**核心功能：**
- 内置MCP服务器可快速启用/禁用
- 用户可通过修改配置文件添加更多服务器
- 开源且用户友好，适合初学者使用
- 未来将持续完善MCP支持

### Apify MCP Tester
[Apify MCP Tester](https://github.com/apify/tester-mcp-client) 是通过Server-Sent Events (SSE)连接任意MCP服务器的开源客户端。该独立Apify Actor专为测试SSE协议下的MCP服务器设计，支持Authorization头部验证。采用传统JavaScript编写，可在Apify平台直接运行无需配置。

**核心功能：**
- 通过SSE连接任意MCP服务器
- 与[Apify MCP Server](https://apify.com/apify/actors-mcp-server)协同工作，对接多个Apify [Actors](https://apify.com/store)
- 根据上下文和用户查询动态调用工具（需服务器支持）

### BeeAI Framework
[BeeAI Framework](https://i-am-bee.github.io/beeai-framework) 是用于大规模构建、部署和服务智能体工作流的开源框架。框架内置**MCP工具**原生功能，简化MCP服务器与智能体工作流的集成。

**核心功能：**
- 无缝集成MCP工具到智能体工作流
- 快速实例化来自MCP客户端的框架原生工具
- 计划未来支持智能体MCP能力

**了解更多：**
- [智能体工作流中使用MCP工具的示例](https://i-am-bee.github.io/beeai-framework/#/typescript/tools?id=using-the-mcptool-class)

### Claude Code
Claude Code是Anthropic推出的交互式智能编码工具，通过自然语言指令加速编码过程。支持MCP服务器集成（提示词与工具），同时自身也可作为MCP服务器与其他客户端交互。

**核心功能：**
- 支持MCP服务器的工具与提示词
- 通过MCP服务器提供自有工具供其他客户端调用

### Claude桌面应用
Claude桌面应用提供完整的MCP支持，实现与本地工具和数据源的深度集成。

**核心功能：**
- 完整资源支持，可附加本地文件和数据
- 提示词模板支持
- 工具集成支持命令和脚本执行
- 本地服务器连接保障隐私安全

> ⓘ 注意：Claude.ai网页应用暂不支持MCP，相关功能仅限桌面版。

### Cline
[Cline](https://github.com/cline/cline) 是VS Code中的自主编码智能体，可在逐步确认后执行文件编辑、命令运行、浏览器操作等任务。

**核心功能：**
- 通过自然语言创建添加工具（如"添加网页搜索工具"）
- 通过`~/Documents/Cline/MCP`目录共享Cline创建的MCP服务器
- 可视化展示MCP服务器配置及其工具、资源与错误日志

### Continue
[Continue](https://github.com/continuedev/continue) 是开源AI代码助手，原生支持全部MCP功能。

**核心功能**
- 输入"@"引用MCP资源
- 提示词模板显示为斜杠命令
- 直接在聊天中使用内置及MCP工具
- 支持VS Code和JetBrains IDE，兼容任意大模型

### Copilot-MCP
[Copilot-MCP](https://github.com/VikashLoomba/copilot-mcp) 通过MCP提供AI编程辅助。

**核心功能：**
- 支持MCP工具与资源
- 集成开发工作流
- 可扩展AI能力

### Cursor
[Cursor](https://docs.cursor.com/advanced/model-context-protocol) 是AI代码编辑器。

**核心功能**：
- 在Cursor Composer中支持MCP工具
- 同时支持STDIO和SSE协议

### Daydreams
[Daydreams](https://github.com/daydreamsai/daydreams) 是用于链上执行的生成式智能体框架。

**核心功能：**
- 配置中支持MCP服务器
- 开放MCP客户端接口

### Emacs Mcp
[Emacs Mcp](https://github.com/lizqwerscott/mcp.el) 是为连接MCP服务器设计的Emacs客户端，支持与[gptel](https://github.com/karthink/gptel)和[llm](https://github.com/ahyatt/llm)等AI插件的工具调用集成，遵循Emacs标准工具调用格式。

**核心功能：**
- 为Emacs提供MCP工具支持

### fast-agent
[fast-agent](https://github.com/evalstate/fast-agent) 是Python智能体框架，通过声明式语法创建支持多模态的智能体工作流，完整兼容Anthropic和OpenAI模型。

**核心功能：**
- 基于MCP原生类型的PDF和图像支持
- 含穿透测试和回放模拟器的交互式开发前端
- 内置"Building Effective Agents"工作流支持
- 可将智能体部署为MCP服务器

### Genkit
[Genkit](https://github.com/firebase/genkit) 是跨语言GenAI应用开发SDK。[genkitx-mcp](https://github.com/firebase/genkit/tree/main/js/plugins/mcp)插件支持将Genkit工具和提示词转换为MCP服务器，或作为客户端连接MCP服务器。

**核心功能：**
- 支持工具和提示词的客户端功能（资源部分支持）
- Genkit开发UI中的丰富服务发现功能
- 与现有Genkit工具/提示词无缝互操作
- 兼容主流GenAI供应商的多款模型

### GenAIScript
使用[GenAIScript](https://microsoft.github.io/genaiscript/)（JavaScript）以编程方式组合大模型提示词，协调大模型、工具和数据。

**核心功能：**
- JavaScript提示词工具箱
- 提升开发效率的抽象层
- 与Visual Studio Code深度集成

### Goose
[Goose](https://github.com/block/goose) 是通过自动化编码任务增强开发效率的开源AI智能体。

**核心功能：**
- 通过工具暴露MCP功能
- 可通过[扩展目录](https://block.github.io/goose/v1/extensions/)、CLI或UI直接安装MCP
- 支持[构建自定义MCP服务器](https://block.github.io/goose/docs/tutorials/custom-extensions)扩展功能
- 内置开发、网页抓取、自动化、记忆等功能工具，支持JetBrains和Google Drive集成

### LibreChat
[LibreChat](https://github.com/danny-avila/LibreChat) 是支持多AI供应商的开源可定制聊天UI，现已集成MCP。

**核心功能：**
- 通过MCP服务器扩展现有工具生态（含[代码解释器](https://www.librechat.ai/docs/features/code_interpreter)和图像生成工具）
- 为可定制[智能体](https://www.librechat.ai/docs/features/agents)添加工具，兼容主流大模型
- 开源可自托管，支持安全多用户体系
- 路线图包含扩展MCP功能支持

### mcp-agent
[mcp-agent] 是基于模型上下文协议构建智能体的简易组合框架。

**核心功能：**
- MCP服务器自动连接管理
- 向大模型暴露多服务器工具
- 实现[Building Effective Agents](https://www.anthropic.com/research/building-effective-agents)定义的全部模式
- 支持工作流暂停/恢复信号（如等待人工反馈）

### Microsoft Copilot Studio
[Microsoft Copilot Studio] 是构建定制化AI应用和智能体的SaaS平台，赋能开发者创建部署复杂AI解决方案。

**核心功能：**
- 支持MCP工具
- 通过MCP服务器扩展Copilot Studio智能体
- 利用微软统一治理的安全API管理方案

### OpenSumi
[OpenSumi](https://github.com/opensumi/core) 是快速构建AI原生IDE产品的框架。

**核心功能：**
- 在OpenSumi中支持MCP工具
- 兼容内置IDE MCP服务器和自定义服务器

### oterm
[oterm] 是Ollama终端客户端，支持创建带工具的聊天/智能体。

**核心功能：**
 - 支持多可定制聊天会话连接Ollama工具
 - 兼容MCP工具

### Roo Code
[Roo Code](https://roocode.com) 通过MCP提供AI编程辅助。

**核心功能：**
- 支持MCP工具与资源
- 集成开发工作流
- 可扩展AI能力

### Sourcegraph Cody

**核心功能：**

### SpinAI

**核心功能：**

### Superinterface

**核心功能：**
- 通过React组件或脚本标签嵌入使用MCP服务器工具

- 兼容任意AI供应商模型（OpenAI/Anthropic/Ollama等）

### TheiaAI/TheiaIDE
[Theia AI](https://eclipsesource.com/blogs/2024/10/07/introducing-theia-ai/) 是构建AI增强工具和IDE的框架。[AI驱动的Theia IDE](https://eclipsesource.com/blogs/2024/10/08/introducting-ai-theia-ide/) 是基于Theia AI构建的开放灵活开发环境。

**核心功能：**
- **工具集成**：Theia AI支持智能体（含Theia IDE）通过MCP服务器实现工具交互
- **可定制提示词**：Theia IDE支持定义适配提示词，动态集成MCP服务器
- **自定义智能体**：支持创建利用MCP能力的定制智能体，实时设计专属工作流

Theia AI与Theia IDE的MCP集成提供高度灵活性，是探索适配MCP的强大平台。

**了解更多：**
- [Theia IDE与Theia AI支持MCP公告](https://eclipsesource.com/blogs/2024/12/19/theia-ide-and-theia-ai-support-mcp/)

**核心功能：**

- 与VS Code扩展系统集成注册MCP服务器

### Windsurf Editor

### Zed

- 增强编码工作流的工具集成

- 暂不支持MCP资源

## 为应用添加MCP支持

如果您已为应用添加MCP支持，欢迎提交PR加入本列表。MCP集成能为用户提供强大的情境化AI能力，并使您的应用成为MCP生态的一部分。

添加MCP支持的优势：
- 允许用户自带上下文和工具
- 加入持续增长的互操作AI应用生态
- 提供灵活的集成选项
- 支持本地优先的AI工作流

可通过[Python SDK文档](https://github.com/modelcontextprotocol/python-sdk)或[TypeScript SDK文档](https://github.com/modelcontextprotocol/typescript-sdk)开始MCP集成。
