Model Context Protocol 采用基于字符串的版本标识符，格式为 `YYYY-MM-DD`，用于表示最后一次进行不向后兼容变更的日期。

<Info>The protocol version will _not_ be incremented when the
protocol is updated, as long as the changes maintain backwards compatibility. This allows
for incremental improvements while preserving interoperability.</Info>

## 版本修订

修订版本可能被标记为：

- **草案（Draft）**：进行中的规范，尚未准备就绪供使用。
- **现行（Current）**：当前协议版本，已可供使用并可能继续接收向后兼容的变更。
- **终版（Final）**：已完成的过往规范，将不再进行修改。

当前协议版本为 [**2025-03-26**](/specification/2025-03-26/)。

## 版本协商

版本协商发生在[初始化阶段](/specification/2025-03-26/basic/lifecycle#initialization)。客户端和服务端**可以**同时支持多个协议版本，但**必须**协商确定会话使用的单一版本。

当版本协商失败时，协议会提供适当的错误处理机制，允许客户端在无法找到与服务器兼容的版本时优雅地终止连接。
